# 华南师范大学计算机专业课程管理系统 - 项目需求文档

## 一、项目概述

### 1.1 项目背景

华南师范大学计算机专业课程体系涵盖数据结构、算法设计、操作系统、计算机网络、数据库原理等核心课程。学生在学习过程中面临知识体系庞大、学习资源分散、个性化需求多样等挑战。传统学习方式难以满足学生随时随地获取精准知识支持的需求。

本项目旨在开发一套面向华南师范大学计算机专业学生的课程管理系统，集成课程信息管理、电子资源访问、智能知识检索及多智能体协同对话等功能，提升学生学习体验与专业知识获取效率。

### 1.2 系统总体目标

1. **课程信息管理**：提供已选课程展示、课程详情查看、课程资料下载等功能
2. **电子资源整合**：构建电子书库，支持按课程关联、专业分类、关键词等多维度检索
3. **智能知识检索**：基于RAG架构实现专业知识的精准检索与流式回答
4. **多智能体协同**：采用多Agent架构，实现课程咨询、代码解答、电子书资讯等多场景对话
5. **个性化学习**：支持学习行为跟踪、知识点关联推荐，助力构建完整知识体系

### 1.3 目标用户

| 用户类型 | 使用场景 | 核心需求 |
|---------|---------|---------|
| 计算机专业学生 | 课程学习、作业辅助、知识巩固 | 获取课程信息、查阅电子资源、智能问答、代码解答 |
| 课程助教 | 答疑辅助、学习指导 | 检索知识点、生成标准答案参考 |
| 课程教师 | 备课参考、教学辅助 | 管理课程资料、分析学生学习情况 |

## 二、核心功能模块详细规范

### 2.1 课程管理模块

#### 2.1.1 功能范围

仅提供课程信息的展示和查询功能，不提供课程选修/退选功能。

#### 2.1.2 功能点详解

**(1) 已选课程列表展示**
- 展示当前学生已选课程列表
- 包含信息：课程名称、教师信息（姓名、职称）、学分、上课时间、上课地点、课程状态
- 支持按学期、课程类型（必修/选修）筛选

**(2) 课程详情页**
- 课程基本信息：课程代码、名称、学分、学时、开课学期
- 课程大纲：课程简介、教学目标、教学内容概述
- 教学进度：周次、教学内容、作业安排
- 考核方式：考勤占比、作业占比、实验占比、期末考试占比
- 教师信息：姓名、职称、联系方式、研究方向

**(3) 课程资料查看与下载**
- 课件列表：PPT、PDF讲义
- 补充材料：参考论文、案例分析、实践指导
- 作业要求：作业题目、截止时间、提交方式
- 支持在线预览和下载功能

**(4) 课程搜索功能**
- 支持按课程名称关键词检索
- 支持按教师姓名检索
- 支持按课程编号检索
- 支持按开课学期筛选

#### 2.1.3 权限控制

- 学生仅能查看本人已选课程信息
- 教师可查看自己授课的课程信息
- 助教可查看协助教学的课程信息

---

### 2.2 电子书库模块

#### 2.2.1 核心功能

实现电子书籍的展示和查询功能，为学生提供丰富的学习资源。

#### 2.2.2 检索方式

**(1) 按课程关联检索**
- 显示特定课程推荐的电子书籍
- 显示与课程内容相关的参考书籍
- 支持从课程详情页直接跳转相关书籍

**(2) 按专业分类检索**
- 数据结构与算法
- 操作系统
- 计算机网络
- 数据库原理
- 人工智能
- 软件工程
- 编程语言（C/C++、Java、Python等）

**(3) 关键词检索**
- 书名检索
- 作者检索
- ISBN检索
- 内容摘要检索

#### 2.2.3 书籍信息展示

- 封面图片
- 书名、副标题
- 作者、译者
- 出版社、出版日期
- ISBN编号
- 内容简介
- 目录结构
- 推荐课程关联

---

### 2.3 RAG专业知识检索模块

#### 2.3.1 知识库构建

**(1) 知识文档范围**
- 数据结构：线性表、树、图、排序算法、查找算法
- 算法设计：贪心算法、动态规划、回溯算法、分治算法
- 操作系统：进程管理、内存管理、文件系统、设备管理
- 计算机网络：TCP/IP协议、HTTP协议、网络安全
- 数据库原理：关系模型、SQL语言、事务管理、索引优化

**(2) 结构化处理**
- 知识点分层：章→节→知识点
- 关键概念标注
- 公式和代码块标记
- 知识点关联关系建立

#### 2.3.2 用户体验要求

**(1) 流式输出**
- 实现打字机效果
- 逐字逐句显示回答内容
- 支持回答过程中的暂停和继续

**(2) 响应时间**
- 查询响应时间≤2秒
- 流式输出首字符延迟≤1秒
- 回答完整输出时间≤10秒

#### 2.3.3 检索准确性

- 检索结果与查询问题相关度≥85%
- 引用来源清晰标注
- 支持知识点溯源

---

### 2.4 多智能体协同与对话模块

#### 2.4.1 架构设计

**(1) 协调Agent (Coordination Agent)**
- 负责接收用户请求
- 分析请求意图
- 路由到合适的Agent
- 汇总Agent返回结果
- 生成最终响应

**(2) Agent职责配置**

| Agent名称 | 职责描述 | 适用场景 |
|-----------|---------|---------|
| 理解Agent | 用户意图识别、关键信息提取 | 问题理解与分析 |
| 检索Agent | 知识检索、数据库查询 | 检索课程、书籍、知识库 |
| 推理Agent | 知识推理与综合 | 逻辑推理、多角度分析 |
| 生成Agent | 答案生成、格式化输出 | 流式生成、引用标注 |
| 验证Agent | 答案质量验证 | 准确性检查、质量评分 |
| 推荐Agent | 相关内容推荐 | 知识推荐、学习路径 |

**(3) 技术实现**

- **LangGraph框架应用**
  - 状态图编排多Agent协作流程
  - 支持Agent间消息传递
  - 实现决策路由逻辑

- **会话持久化**
  - 对话历史存储到数据库
  - 支持跨会话上下文恢复
  - 支持对话历史导出

- **流式输出与推理可见性**
  - 通过SSE(Server-Sent Events)技术实现
  - 分阶段推送中间推理过程

#### 2.4.2 推理过程展示

**(1) 第一阶段：理解分析**
- 显示问题理解过程
- 展示提取的实体和关键词
- 可视化意图分类结果

**(2) 第二阶段：检索执行**
- 显示Agent处理状态
- 展示知识检索过程
- 显示数据库查询结果

**(3) 第三阶段：汇总结果**
- 呈现最终回答
- 标注引用来源
- 提供相关知识点推荐

### 2.5 LLM模式设置

#### 2.5.1 模式类型

**(1) 知识检索模式**
- 专注于专业知识查询与解答
- 严格基于知识库内容生成回答
- 标注引用来源
- 适合课程知识点查询、作业辅导

**(2) 对话模式**
- 支持日常学习交流与咨询
- 开放域对话
- 更自然的语言风格
- 适合学习计划讨论、学习方法交流

#### 2.5.2 切换机制

- 提供模式切换按钮
- 清晰的模式标识（图标+文字）
- 平滑切换，不丢失对话上下文

#### 2.5.3 模式特性对比

| 特性 | 知识检索模式 | 对话模式 |
|------|------------|---------|
| 回答来源 | 严格基于知识库 | 开放域生成 |
| 引用标注 | 显示来源 | 无 |
| 语言风格 | 正式、严谨 | 自然、亲切 |
| 适用场景 | 知识点查询、作业辅导 | 学习交流、规划讨论 |

## 四、系统架构设计

### 4.1 架构风格

采用领域驱动设计（DDD）分层架构：

- **领域层**：定义业务实体和仓储接口
- **应用层**：编排业务用例，协调领域对象
- **基础设施层**：实现仓储接口，提供LLM客户端和数据库连接
- **接口层**：处理HTTP请求响应，提供前端界面

### 4.2 技术栈

| 层次 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Vue3 | 3.4+ |
| 状态管理 | Pinia | 2.1+ |
| UI框架 | Tailwind CSS | 3.4+ |
| 后端框架 | FastAPI | 0.115+ |
| Agent框架 | LangGraph | 1.x |
| 向量数据库 | Chroma | 0.5+ |
| 业务数据库 | TiDB Cloud | MySQL兼容 |
| LLM接口 | SiliconFlow (OpenAI兼容) | - |

### 4.3 数据流设计

```
前端请求 → API网关 → 请求类型路由
    │
    ├── 课程查询 → 课程服务
    ├── 书籍检索 → 电子书库服务
    ├── 知识问答 → 问答服务 → 多Agent协调
    │                    │
    │                    ├── 理解Agent
    │                    ├── 检索Agent
    │                    ├── 生成Agent
    │                    └── 验证Agent
    └── 用户认证 → 认证服务
```
